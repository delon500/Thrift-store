1. HIGH-LEVEL OVERVIEW

This repository is a three-part JavaScript application:

- frontend: customer-facing Vite + React application.
- admin: administrator Vite + React application.
- backend: Express API server using PostgreSQL, JWT authentication, Cloudinary uploads, and OpenAI product image analysis.

There is no root package.json. Each application is managed independently from its own folder.

8. AUTHORIZATION MODEL

JWT tokens are generated in backend/controllers/authController.js.
The token payload includes:
- id
- role
- institution_id
- status

Middleware:
- protect: requires Authorization: Bearer <token>.
- allowRoles(...roles): restricts protected routes by req.user.role.

11. NOTABLE OBSERVATIONS

- There is no root workspace setup, so frontend, admin, and backend must be installed and run separately.
- The root .gitignore currently ignores .idea, .vscode, .env, and AGENTS.md, but it does not list node_modules. If node_modules is not globally ignored, it can accidentally be committed.
- The backend uses environment variables but no .env.example file is present.
- The backend references PostgreSQL tables, but no schema or migration directory was found.
- backend/controllers/authController.js uses institution_id in registerStudentParent without destructuring it from req.body, which can break student/parent registration.
- frontend/src/features/auth/api/authApi.js defines registerInstitution without a formData parameter but uses formData inside the function.
- Product image uploads are handled by multer disk storage and uploaded to Cloudinary, but temporary local upload cleanup is not visible in the scanned code.
- backend/package.json lists nodemon under dependencies instead of devDependencies.
- Both frontend and admin hard-code http://localhost:5000/api instead of reading the API URL from environment configuration.

13. SUMMARY

The project is organized as a split full-stack ecommerce/thrift platform:

- frontend serves students, parents, and general users.
- admin serves administrators managing users, institutions, products, and lost/found items.
- backend exposes the shared API, handles authentication, connects to PostgreSQL, uploads images through Cloudinary, and uses OpenAI to analyze product photos.

The folder layout is understandable and feature-based in both React apps. The main missing project-level pieces are root orchestration, database schema/migrations, environment documentation, and a few small API implementation fixes.
